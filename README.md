### Project Structure

Contains both Selenium Test NG framework along with Selenium Cucumber TestNg framework style both in one project itself.

---

#### CLI Commands to Trigger Run

Selenium TestNg
```bash
mvn clean test -Dsurefire.suiteXmlFiles=order-tools-suite-run.xml
```
Selenium Cucumber
```bash
mvn test -Dtest=OrderToolsTestRunner.java
```
---

### Architecture

#### Selenium Test NG

- Page Object Module Architecture.
- Utils classes for Reusability of utility functions.
- Custom Well Readable Extent Reports with Thread safe usage for parallel execution
- Listeners (Hook Managemnt) for working with custom report generation
- testng_suite_xml.xml file to run suite


#### Selenium Cucumber

- Page Object Module Architecture.
- Utils classes for Reusability of utility functions.
- Feature files and Step Definition classes as per POM modules
- PICO containers for dependency injection plugin for sharing data between step definition classes.
- Custom Well Readable Extent Reports with Thread safe usage for parallel execution
- Hooks File for working with custom report generation
- CucumberTestNG Runner file to run suite

### Reports

#### [1] Cucumber Default Report

- path : `cucumber_reports/default_cucumber_report`
- desc : using cucumber report
- how configured :
  - In Runner - `html:cucumber_reports/default_cucumber_report/cucumber.html`
- comes with cucumber
- how it works : Automatically


#### [2] Extent Report Generated By Cucumber

- path : `cucumber_reports/extent_reports`
- how configured :
  - In Runner - `com.aventstack.extentreports.cucumber.adapter.ExtentCucumberAdapter`
- And - `src/test/resources/extent.properties`
- External Library added


#### [3] Custom Extent Report

- path : `cucumber_reports/custom-extent-reports`
- how configured :
  - In Hooks File
  - Thread Safe Extent Test Blocks created in Hooks and Accessed in StepDefination Files
  - External Library Added
  - Report fully manually built for neat data reporting
